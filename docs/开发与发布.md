# 开发与发布

本文记录这个 fork 的日常开发和发布约定。目标很简单：围绕 Windows、Visual Studio、Win32 和 DX11 保持构建路径单一、验证口径清晰。

## 开发基线

当前默认基线：

- 平台：Windows
- 工程：`EUI-NEO-DX11.slnx` / `EUI-NEO-DX11.vcxproj`
- 语言标准：C++20
- 入口：`core/app/win32_app_main.cpp`
- 渲染后端：DX11

## 本地构建

推荐直接使用 Visual Studio：

1. 打开 `EUI-NEO-DX11.slnx`
2. 选择 `Debug` 或 `Release`
3. 选择 `Win32` 或 `x64`
4. 构建并运行

命令行示例：

```powershell
msbuild .\EUI-NEO-DX11.vcxproj /p:Configuration=Release /p:Platform=x64
```

## 改动验证

常规改动至少验证：

- `Debug` 与 `Release` 至少其一能完整编译
- 文本、图标、图片、滚动、IME 没有明显回归
- 窗口 resize、最小化、恢复、多窗口和托盘逻辑正常
- DX11 路径下 dirty rect、blur 和 layer 没有明显残影或漏刷

建议优先打开仓库内的示例或应用页面做人工回归，而不是依赖旧上游的多后端矩阵。

## 文档要求

文档、站点和示例说明必须与当前 fork 一致：

- 统一采用 Win32 + DX11 + C++20 口径
- 以 Visual Studio 工程作为主要工作流
- 不再沿用上游旧的多后端叙事


## 提交建议

提交信息保持清晰即可，推荐继续使用带 scope 的 Conventional Commits：

```text
type(scope): subject
```

常见类型：

- `feat`
- `fix`
- `docs`
- `refactor`
- `perf`
- `test`
- `build`
- `chore`

## 发布约定

发布前至少确认：

- 工程能在目标配置下完整构建
- `assets/` 资源与可执行文件一同可用
- README、站点和关键文档没有残留旧上游定位
- DX11 渲染路径下的主要页面能正常运行

这个 fork 当前应被视为 Windows 定向分支。发布说明、截图、性能数据和构建步骤，都应以这一定位为准。
